## 📝 Про проєкт
**mGit** — це легковагова, ізольована консольна система контролю версій (утиліта командного рядка), створена в рамках навчальної практики за варіантом підвищеної складності **Б-5**. 

Утиліта дозволяє повністю контролювати життєвий цикл текстових файлів: створювати знімки стану (snapshots), відстежувати історію змін за допомогою логування, аналізувати покрокові розбіжності за допомогою вбудованого **diff-алгоритму** з кольоровою підсвіткою та миттєво відкочувати файли до будь-якої збереженої версії.

> 🛡️ **Безпека понад усе:** Утиліта повністю захищена від аварійних завершень (`ValueError`, `IndexError`, `FileNotFoundError`). При будь-якому некоректному вводі програма виводить інформативне TUI-повідомлення, зберігаючи стабільність репозиторію.

---

## ✨ Ключовий функціонал

*   `init` — Ініціалізація локального прихованого сховища `.mgit/`.
*   `snapshot` — Фіксація поточної версії файлу з додаванням унікального ID, мітки часу та кастомного коментаря.
*   `log` — Виведення структурованої, красивої TUI-таблиці з ретроспективою всіх версій.
*   `diff` — Порядковий аналіз розбіжностей між двома знімками за допомогою інтегрованого алгоритму порівняння.
*   `rollback` — Безпечний відкат оригінального файлу до стану будь-якого збереженого знімка.

---

## ⚙️ Архітектура та формат даних
Дані між запусками програми надійно фіксуються в локальній файловій системі:
*   `.mgit/snapshots/` — Директорія-архів, де зберігаються точні копії версій файлів.
*   `.mgit/index.json` — База даних метаданих у форматі JSON (кодування UTF-8).

### Приклад структури індексу:
```json
{
    "snapshots": [
        {
            "id": 1,
            "filename": "snap_1_training_plan.txt",
            "original_path": "/home/user/projects/training_plan.txt",
            "comment": "Базова програма на мезоцикл",
            "timestamp": "2026-06-24 14:20:00"
        }
    ],
    "latest_id": 1
}

```

## 🚀 Швидкий запуск «з чистого аркуша»

### 1. Клонування та підготовка середовища

```bash
git clone [https://github.com/ТВІЙ_НІКНЕЙМ/mgit.git](https://github.com/ТВІЙ_НІКНЕЙМ/mgit.git)
cd mgit
pip install -r requirements.txt

```

### 2. Ініціалізація сховища

```bash
python mgit.py init

```

*Вивід у термінал:*

`[bold green]Успіх:[/bold green] Створено порожній репозиторій mGit.`

## 📸 Демонстрація роботи (Тест-кейси)

### Крок 1. Фіксація першої версії файлу

Створимо файл і зробимо перший знімок:

```bash
echo "Ранкове тренування: Кардіо 5 км" > спорт.txt
python mgit.py snapshot спорт.txt "Перша версія плану"

```

*Вивід:* `[bold green]Знімок #1 успішно створено![/bold green] (Перша версія плану)`

### Крок 2. Модифікація та другий знімок

Змінимо вміст файлу, додавши нові таски, та знову зафіксуємо стан:

```bash
echo -e "Ранкове тренування: Кардіо 5 км\nВечірнє тренування: Робота на татамі, 5 спарингів" > спорт.txt
python mgit.py snapshot спорт.txt "Додав вечірній блок"

```

*Вивід:* `[bold green]Знімок #2 успішно створено![/bold green] (Додав вечірній блок)`

### Крок 3. Перегляд історії (`log`)

```bash
python mgit.py log

```

🎨 *Програма згенерує красиву таблицю:*

```text
 📜 Історія знімків mGit
 ┌────┬─────────────────────┬───────────┬──────────────────────┐
 │ ID │ Дата/Час            │ Файл      │ Коментар             │
 ├────┼─────────────────────┼───────────┼──────────────────────┤
 │ 1  │ 2026-06-24 22:10:05 │ спорт.txt │ Перша версія плану   │
 │ 2  │ 2026-06-24 22:15:30 │ спорт.txt │ Додав вечірній блок  │
 └────┴─────────────────────┴───────────┴──────────────────────┘

```

### Крок 4. Аналіз розбіжностей (`diff`)

Порівняємо знімок #1 та знімок #2, щоб побачити точні зміни у коді/тексті:

```bash
python mgit.py diff 1 2

```

🔥 *Кольоровий TUI-вивід у терміналі:*

> `Ранкове тренування: Кардіо 5 км`
> `[green]+ Вечірнє тренування: Робота на татамі, 5 спарингів[/green]`

### Крок 5. Миттєвий відкат (`rollback`)

Якщо вечірнє тренування скасувалося і нам треба повернути попередній стан файлу:

```bash
python mgit.py rollback 1

```

*Вивід:* `[bold green]Успіх:[/bold green] Файл відновлено до стану знімка #1!`

## 🛠️ Обробка некоректного вводу (Фазове тестування)

Проєкт повністю протестований на стійкість до відмов. Програма **ніколи не падає з критичною помилкою (`traceback`)**:

* **Введення літер замість ID:** `python mgit.py rollback abc`
*Результат:* `[bold red]Помилка типу даних:[/bold red] ID знімків мають бути цілими числами.`
* **Спроба викликати неіснуючий ID:** `python mgit.py rollback 999`
*Результат:* `[bold red]Помилка:[/bold red] Знімок з ID 999 не знайдено.`
* **Спроба зробити знімок відсутнього файлу:** `python mgit.py snapshot noname.txt "test"`
*Результат:* `[bold red]Помилка:[/bold red] Файл 'noname.txt' не знайдено.`
